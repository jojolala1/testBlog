# TypeScript vs JavaScript

JavaScript est l'un des langages de programmation les plus populaires au monde. Mais depuis quelques années, TypeScript, une surcouche de JavaScript, gagne en popularité. Alors, lequel choisir pour ton projet ? Explorons les différences.

## Qu'est-ce que JavaScript ?

JavaScript est un langage de programmation interprété, principalement utilisé pour le développement web. Il est léger, flexible et fonctionne directement dans les navigateurs.


```js
const add = (a,b) =>a + b;

console.log(add(1,2));
```
### Avantages de JavaScript :
- Simple à apprendre et à utiliser.
- Large écosystème et communauté.
- Compatible avec tous les navigateurs modernes.

### Inconvénients de JavaScript :
- Pas de typage statique, ce qui peut entraîner des bugs difficiles à détecter.
- Moins adapté pour les projets complexes.

## Qu'est-ce que TypeScript ?

TypeScript est un sur-ensemble de JavaScript développé par Microsoft. Il ajoute un typage statique et d'autres fonctionnalités avancées à JavaScript.

### Avantages de TypeScript :
- Typage statique, réduisant les erreurs au moment de la compilation.
- Meilleure lisibilité et maintenabilité du code.
- Support natif pour les IDE avec autocomplétion et vérification des types.

### Inconvénients de TypeScript :
- Courbe d'apprentissage plus élevée pour les débutants.
- Nécessite une étape de compilation pour convertir en JavaScript.

## Quand utiliser TypeScript ou JavaScript ?

- **JavaScript** : idéal pour les petits projets ou les scripts rapides.
- **TypeScript** : recommandé pour les projets complexes ou à long terme nécessitant une grande maintenabilité.

## Conclusion

Le choix entre TypeScript et JavaScript dépend de tes besoins. Si tu débutes ou travailles sur un petit projet, JavaScript est un excellent choix. Pour des projets plus complexes, TypeScript peut t'aider à écrire un code plus robuste et maintenable.
